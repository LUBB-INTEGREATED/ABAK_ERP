# UI/UX Audit — 2026-05-22

Senior-designer audit of the ABAK ERP web app, grounded in code review (not
screenshots). Findings cite `file:line` and are organised by systemic root
cause, not screen-by-screen. Companion to `DESIGN_SYSTEM_MASTER.md`.

## Verdict

The product **works** — flows are correct, components are real, Norman
primitives are in the right places. But visually it reads as "shadcn defaults
plus brand colour sprinkled on top." That's the gap: it doesn't yet look or
feel like a designed system. It looks like a wireframe that someone forgot to
design.

The good news: every issue below is systemic — token-level fixes that
propagate across the whole product. We're not repainting screens one-by-one.

---

## The 7 systemic issues

### 1. Two competing color systems

`packages/web/tailwind.config.ts:15-46` defines `abak-blue` and `abak-gold` as
hex scales. `packages/web/src/app/globals.css:6-30` defines a parallel
`--primary` / `--secondary` / `--accent` set in HSL. Components mix both:

- `Button` uses `bg-primary` (token) → CSS-resolves to abak-blue.
- `KpiCard` uses `text-abak-blue` (hex direct) → bypasses the token.
- Same color, two import paths. **Rename "primary" and half the app stays blue.**

This is the single biggest issue. Until every component talks to the same
tokens, refactoring colors is impossible.

### 2. The brand color is everything, so it's nothing

`text-abak-blue` is the H1 color (`dashboard/page.tsx:108`), the CTA color
(`Button` default), the chart-bar color (`dashboard/page.tsx:215`), the focus
ring (`globals.css:--ring`), the sidebar background, every KPI number, every
link, every entity number, every print section header.

When everything is brand, brand stops meaning anything. The eye can't find
primary action because primary action is the same hue as the page title and
the chart bars.

### 3. Semantic colors break their semantics

`tailwind.config.ts:80`:

```ts
success: '#A78B42',  // this is the gold brand color
warning: '#D97706',
error: '#DC2626',
```

A "VIP client" badge (uses brand gold) is now visually indistinguishable from
a "Success" badge. Same hex. The system has no actual green-for-success. When
a Finance Manager looks at "Payment validated" they should feel relief — gold
doesn't say that.

`dashboard/page.tsx:362-367`: KpiCard encodes "highlight" / "danger" / default
as three colors but uses **abak-gold for highlight** — same as VIP and Success.
Three meanings, one color.

### 4. No type scale

- Page H1: `text-2xl font-bold` (24px) — `dashboard/page.tsx:108`
- Card title: `text-base` (16px) — `dashboard/page.tsx:160` — overrides the
  component default everywhere
- Section header inside card: also `text-base font-semibold`
- Body: `text-sm` (14px) almost everywhere
- Muted helper: `text-xs` (12px)

No 36 / 24 / 20 / 16 / 14 / 12 ladder. There's no display style for hero
numbers, no distinct section-header style. Every block of text fights every
other block.

`Card`'s component default is `text-2xl` for `CardTitle` (`card.tsx:46`) and
**nobody uses the default**. Always overridden to `text-base`. The component
default lies.

### 5. Action hierarchy is flat

Every button is `bg-primary` solid blue. "Refresh", "New lead", "Save",
"Convert to Project", "Cancel" — same weight, same color, same size. Norman's
`primary-action` rule (one primary CTA per screen) violated on every list
page.

`leads/page.tsx:201-216`: "Refresh" outline + "New lead" filled, both blue.
Refresh is utility; New Lead is the goal — and they sit at the same visual
weight.

### 6. Surfaces are over-decorated

`card.tsx:11`: every card carries `border` + `shadow-sm` + `rounded-lg`
simultaneously. That's three elevation signals stacked. Either border-only
(flat) or shadow-only (Material elevation) — not both. Result: cards look
heavy and busy, like printed forms rather than software surfaces.

### 7. Filter overload, no progressive disclosure

`leads/page.tsx:246-380`: 9 filter inputs jammed into one wrap row — Search,
Channel, Status, Priority, SLA, Service, Location, Min budget, Max budget,
Date. All same weight. No "Advanced" collapse. New users see this and freeze
(Norman's overwhelm-upfront anti-pattern).

## Fix plan, ranked by impact

### Tier 0 — Token surgery (1–2 days, unblocks everything)

| #   | What                                                                                                                                                                                                   | Where                |
| --- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------- |
| 1   | **Delete the hex `abak-blue` scale** from `tailwind.config.ts`. Replace every `text-abak-blue` / `bg-abak-blue` with `text-primary` / `bg-primary`. Now the token is the only source of truth.         | global               |
| 2   | **Add real semantic tokens**: `--success` = emerald-600, `--warning` = amber-600, `--info` = sky-600. Stop using gold as success.                                                                      | `globals.css`        |
| 3   | **Add a type scale** in tailwind config: `display-lg / display-md / heading-lg / heading-md / heading-sm / body / body-sm / caption`. Map each to a font-size + line-height + tracking pair.           | `tailwind.config.ts` |
| 4   | **Fix Card defaults** in `card.tsx`: drop the `shadow-sm` OR the `border` (pick one). Change `CardTitle` default to `text-base font-semibold` so callers stop overriding it.                           | `card.tsx:11,46`     |
| 5   | **Fix focus-ring contrast on filled buttons** — `--ring` is the same hue as `bg-primary`, so the focus ring is invisible on the primary button. Set ring to `--ring-offset` white + outer dark stroke. | `button.tsx:9`       |

## Shipping options

- **Option A — Token surgery first (recommended).** Tier 0 alone, in one focused
  PR. Two days, every screen improves silently the moment it lands. Then
  Tier 1/2/3 can be done per-screen incrementally.
- **Option B — Pick the 3 hero screens** (Dashboard, Lead detail, Quote builder)
  and redesign them properly first, leaving the rest. Faster visible win, but
  the underlying system stays broken so the new screens drift back.
- **Option C — Full rebuild against a fresh design system.** ~2 weeks.
  Probably overkill at this stage.

Recommendation: do **A**, then **B for the dashboard only** as a showcase. The
dashboard is the screen everyone sees first; it should anchor the new feel.
